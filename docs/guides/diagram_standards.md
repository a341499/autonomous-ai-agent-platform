\# Diagram Standards



\## Purpose



This document defines the standard visual style for architecture and workflow diagrams used throughout the AIGA portfolio.



\---



\## General Principles



All diagrams should:



\- Be clean and uncluttered

\- Use a white background

\- Use consistent typography

\- Follow a top-to-bottom or left-to-right flow

\- Use consistent terminology across all projects



\---



\## Naming Convention



Diagram filenames should use lowercase with underscores.



Examples:



\- system\_architecture.png

\- workflow\_lifecycle.png

\- memory\_architecture.png

\- project\_structure.png



\---



\## Colour Palette



\- Primary: Dark Blue

\- Secondary: Green

\- Accent: Orange

\- Background: White



\---



\## Font



\- Segoe UI

\- Calibri

\- Arial



\---



\## Preferred Size



\- GitHub diagrams: 1600 × 900

\- LinkedIn diagrams: 1920 × 1080
